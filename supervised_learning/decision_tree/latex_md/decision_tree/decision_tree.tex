  Requer pouca preparação de dados. Outras técnicas geralmente requerem
  a normalização de dados e variáveis dummy.
\item
  \textbf{O custo de prever dados é logarítmico.}
\item
  Capaz de lidar com dados numéricos e categóricos.
\item
  Capaz de lidar com problemas de várias saídas. (dimensões no eixo x)
\item
  Possível validar um modelo usando testes estatísticos. Isso torna
  possível explicar a confiabilidade do modelo.
\end{itemize}

    \subsection{disadvantages:}\label{disadvantages}

\begin{itemize}
\tightlist
\item
  \textbf{overfitting}: gera árvores super complexas que não generalizam
  bem os dados.
\item
  NP-completo
\item
  São modelos instáveis (alta variância), pequena variações nos dados de
  treino podem resultar em árvores completamente distintas.
\item
  os algoritmos de árvore de decisão são baseados em algoritmos
  heurísticos, como o algoritmo guloso, em que decisões locais ótimas
  são tomadas em cada nó. Tais algoritmos não podem garantir o retorno
  da árvore de decisão ótima globalmente. Isso pode ser atenuado pelo
  treinamento de várias árvores em um aprendiz conjunto, onde os
  recursos e amostras são amostrados aleatoriamente com a substituição.
\end{itemize}
